Privacy

NeverMiss is built to help you manage follow-ups inside Gmail. This policy is intentionally plain English. If anything here conflicts with how the extension behaves, the extension behavior wins and this document must be updated.

What NeverMiss stores

  • Follow-up reminders you create (timing, status).
  • Deal context you add (stage/value/next step) for a thread/contact.
  • Tracking event metadata needed to show signals (e.g., opened counts and timestamps).

What NeverMiss does not do

  • It does not ask for your Gmail password.
  • It does not sell your data.
  • It does not insert ads into Gmail.

Data handling

  • Some data may be stored locally in your browser to keep the UI fast.
  • Some events may be sent to a backend service to support tracking and sync.
  • NeverMiss aims to minimize stored data and retain only what is necessary for core functionality.

Security

  • The UI is rendered using safe DOM APIs (no injecting user content via innerHTML).
